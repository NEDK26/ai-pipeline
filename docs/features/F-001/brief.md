# F-001: 功能定义简报 — 演示配置开关

**Status**: Drafted
**Author**: feature-discovery
**Date**: 2026-07-03

## 问题背景

当前系统无运行时"演示模式"开关。如需切换演示模式，依赖开发人员修改配置文件并重启服务。运维和 Admin 角色无法在管理后台自行操作，造成以下问题：

- **低效**：每次切换需开发介入 + 部署周期，响应慢
- **风险**：直接修改配置文件容易出错，无审计记录可追溯
- **权限失控**：无角色控制，任何能接触配置文件的人都能改

本 Feature 将开关从静态配置提升为运行时配置，提供管理后台 UI 和 API，并记录每次变更的完整审计轨迹。

## 目标用户

| 角色 | 使用频率 | 关键任务 | 差异 |
|------|---------|---------|------|
| ConfigAdmin | 低频（按需） | 查看当前状态、切换开关 | 可读可写 |
| Operator | 低频（按需） | 查看当前状态 | 只读，修改被 API 拒绝 |
| 无权限用户 | — | — | 菜单不可见，API 返回 403 |

## 目标与成功标准

| 维度 | 目标 | 可观测指标 |
|------|------|-----------|
| 用户结果 | ConfigAdmin 可在 30 秒内完成开关切换 | 操作耗时 |
| 业务结果 | 每次切换有完整审计记录（谁、何时、旧值、新值、原因） | `system_operate_log` 中存在对应记录 |
| 权限控制 | Operator 仅可查看，修改操作被拒绝 | API 返回 403 |
| 数据质量 | 重复提交不产生重复审计记录 | 相同新旧值 → 0 条新增审计 |

## 功能范围

**本期必须交付**:
- GET API：查询当前演示模式状态
- PUT API：修改演示模式状态（含原因字段校验）
- 每次修改写入审计日志（`system_operate_log`）
- 幂等保护：相同值不重复写库、不重复审计
- 权限控制：`system:demo-mode:query` / `system:demo-mode:update`
- 前端页面：Switch 开关 + 原因输入框 + 二次确认弹窗
- 前端权限：Operator 只读视图，无权限用户不可达

**可选能力**：无

**依赖条件**:
- `infra_config` 表存在
- `system_operate_log` 表存在
- `@LogRecord` / bizlog-sdk 框架可用
- 前端 Element Plus、`useMessage()`、`v-hasPermi` 可用
