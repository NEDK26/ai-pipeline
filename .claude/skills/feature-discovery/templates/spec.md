# $feature_id：功能规格

## 1. 用户故事
- 采用"作为 <角色>，我希望 <动作>，从而 <价值>"格式；
- 每条用户故事分配唯一编号，例如 US-001；
- 每条用户故事必须关联至少一个验收用例编号。

## 2. 业务规则
- 规则编号，例如 BR-001；
- 触发条件；
- 允许行为；
- 拒绝行为；
- 提示文案或提示目标；
- 例外情况；
- 对应的验收用例。

## 3. 角色与权限
- 谁可以看；
- 谁可以创建、修改、发布、停止、回滚、删除；
- 数据可见范围；
- 权限不足时的系统行为。

## 4. 状态与状态流转
- 状态定义；
- 初始状态；
- 允许转移；
- 禁止转移；
- 转移触发者；
- 转移前置条件；
- 失败后的状态；
- 回退规则。

## 5. 数据与口径
- 关键字段；
- 字段来源；
- 必填规则；
- 计算口径；
- 刷新时机；
- 空值、异常值与历史数据处理。

## 6. 异常与降级
- 网络或服务失败；
- 重复提交；
- 并发操作；
- 数据不一致；
- 依赖系统不可用；
- 用户中断操作；
- 恢复和重试方式。

## 7. 非功能要求
- 性能；
- 审计；
- 安全；
- 可用性；
- 可观测性；
- 兼容性；
- 数据保留或合规要求。
