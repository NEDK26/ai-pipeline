# $feature_id：验收可追溯矩阵

每条验收用例必须映射到至少一种验证方式。

| 验收用例 | 风险等级 | 验证层 | 自动化位置 / 手工步骤 | 执行状态 | 证据路径 |
|----------|----------|--------|----------------------|----------|----------|

## 验证层分类

- **Backend Unit** — 后端单元测试
- **Backend Integration** — 后端集成测试
- **API Contract** — API 契约测试
- **Frontend Unit** — 前端单元测试
- **E2E** — 浏览器端到端测试
- **Visual Review** — 视觉审查
- **Manual Verification** — 人工验证

## 覆盖规则

- 高风险业务规则必须至少有 Backend Unit 或 Backend Integration 覆盖
- 主用户路径必须至少有 E2E 覆盖
- 高风险 UI 操作必须至少有 E2E 和截图证据
- 仅用截图不能证明业务逻辑正确
- 仅用接口成功不能证明用户流程可完成
- 无法自动化的用例必须写清人工验证步骤和原因
