# 缺陷分类规则

## 类别

每个失败项必须归类为以下之一：

- **Product Gap**：需求、规则或验收标准不明确
- **Backend Defect**：后端业务、接口、权限、状态或数据问题
- **Frontend Defect**：页面、交互、状态展示或错误处理问题
- **Contract Mismatch**：前后端接口或字段口径不一致
- **Test Defect**：测试脚本、测试数据或环境问题
- **Environment Issue**：依赖服务、账号、网络或环境不可用
- **Visual Review Finding**：视觉层级、溢出、一致性或可用性问题

## 记录要求

不要在测试结论中写"有问题待优化"。必须写明：

- 如何复现
- 当前结果
- 期望结果
- 影响范围
- 严重级别（P0/P1/P2/P3）
- 建议回流的处理阶段
- 已保存的证据路径
