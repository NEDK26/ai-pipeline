# 分层验证范围定义

## A. 后端测试

验证：
- 业务规则
- 状态流转
- 权限
- 参数校验
- 幂等与重复提交
- 并发或条件更新
- 事务边界
- 错误码
- 审计记录

优先运行项目已有的单元测试和集成测试。若发现关键规则无测试覆盖，可以新增测试代码，但不得修改业务实现使其通过。

## B. 前端测试

验证：
- 表单校验
- 按钮禁用和提交中状态
- 权限不足状态
- 空数据、加载、错误状态
- 接口错误提示
- 高风险操作确认
- 关键状态展示

优先复用项目现有测试框架与组件测试模式。

## C. API 契约与联调验证

验证：
- 请求字段
- 返回字段
- 分页、筛选、排序
- 业务错误码
- 权限限制
- 状态限制
- 字段口径
- 前后端对异常响应的处理一致性

## D. 浏览器端到端验证

若项目已安装 `webapp-testing`，使用它协助：
- 启动本地前后端服务
- 登录或准备测试数据
- 执行关键用户路径
- 捕获浏览器 Console 错误
- 捕获失败网络请求
- 对关键节点截图
- 在失败时保留可复现证据

若项目已有 Playwright E2E 测试，优先扩展现有测试，而不是只做一次性手工浏览器操作。

关键路径至少覆盖：
- 主流程
- 一条权限或状态拒绝流程
- 一条异常或失败恢复流程
- 一条高风险操作确认流程

## E. 视觉验证

默认只对关键页面做截图证据，不默认开启全量像素级视觉回归。

只有满足以下条件时才建立视觉快照基线：
- 页面内容可稳定复现
- 时间、随机数据、动态广告、实时指标已固定或隐藏
- 运行环境可固定
- 基线截图经过人工审查

截图差异必须人工判断原因。禁止因为"截图不同"就自动更新基线并判定通过。
